## RAGD Integration
- What was connected: direct REST client, BM25/vector calls, RRF composition, ledger decisions, graph symbol reads, traceable CLI.
- What metadata is preserved: chunk ID, path, lines, scores, language, chunk type, symbol, citations; `content_hash` via temp adapter.
- What is traceable: plan, retrieve BM25, retrieve vector, RRF, filters, rerank, confidence, assemble.
- Fallback behavior: RAGD unreachable returns non-zero with a clear error; generation is optional and retrieve-only default.

## Local LLM Support
- Provider strategy: registry with mock and Ollama providers.
- Low-VRAM behavior: current 4 GB class GPU refuses configured 3.8 GB model because it exceeds the 3.5 GB ceiling.
- Timeout and retry behavior: provider request timeout is carried in `GenerateRequest`; retry policy deferred.
- Metrics captured: governor plan and provider health in `llm doctor --json`.
